# Техническое задание WeilShop

## 1. Цель проекта

Разработать адаптивный русскоязычный сайт магазина автохимии `WeilShop` с каталогом товаров, подбором по задачам ухода, базой знаний, личным кабинетом, системой заявок без онлайн-оплаты и административной панелью.

## 2. Формат проекта

Проект разрабатывается как `API-first` платформа, включающая:
- публичный сайт
- backend API
- административную панель

## 2.1. Требования к дизайн-концепции

Визуальное направление проекта: `Tech Clean + Practical Expert`.

Интерфейс должен:
- выглядеть современно и технологично
- быть понятным для пользователя без опыта в автохимии
- строиться вокруг задач ухода и сценариев применения
- подчеркивать экспертность и аккуратность бренда

Базовые визуальные принципы:
- светлый спокойный фон
- графитовый основной текст
- глубокий зеленый как основной акцент
- оранжевый для `CTA`
- чистая модульная сетка
- карточки без визуального шума
- высокий приоритет мобильной версии

## 4. Заявки

Сайт не использует онлайн-оплату и оформление доставки на первом этапе.

Оформление происходит через заявку:
- пользователь добавляет товары в корзину
- отправляет заявку
- заявка попадает администратору
- администратор меняет статус заявки
- пользователь видит статус в личном кабинете

Статусы заявки:
- `new`
- `in_review`
- `confirmed`
- `processing`
- `completed`
- `cancelled`

## 5. Авторизация

Регистрация и вход доступны:
- по `email`
- по номеру телефона

Система должна поддерживать:
- хранение пароля в безопасном виде
- подтверждение email
- архитектурную готовность к подтверждению телефона
- восстановление доступа

## 8. Нефункциональные требования

- SEO-friendly структура
- ЧПУ URL
- высокая скорость загрузки страниц
- безопасная авторизация
- масштабируемая архитектура

- уникальные `meta title` и `meta description`

- хлебные крошки
- возможность ручного управления SEO-метаданными из админки
