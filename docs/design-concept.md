# Дизайн-концепция WeilShop

## 1. Рекомендуемое направление

Для `WeilShop` рекомендуется дизайн-подход `Tech Clean + Practical Expert`.

Это сочетание:
- современного технологичного визуального языка
- понятной экспертной подачи
- удобного сценарного выбора автохимии

Сайт должен выглядеть не как шаблонный маркетплейс, а как удобный помощник по уходу за автомобилем, который помогает выбрать правильное средство под конкретную задачу.

## 2. Ключевая идея

Основной акцент делается не на демонстрацию большого количества товаров, а на ответ на вопрос пользователя:

`Что именно нужно сделать с автомобилем и каким средством это лучше решить?`

Поэтому визуальная подача должна строиться вокруг:
- задач ухода
- пошаговых сценариев
- понятных инструкций
- связанных товаров
- доверия к экспертности магазина

## 3. Визуальный характер

Интерфейс должен создавать ощущение:
- чистоты
- технологичности
- аккуратности
- уверенности
- практической пользы

Нужно избегать:
- визуального шума
- перегруженных карточек
- слишком агрессивного дизайна
- дешевого шаблонного e-commerce вида

## 4. Цветовая система

Рекомендуемая палитра:
- основной фон: светлый теплый серый или молочный
- основной текст: графитовый
- вторичный текст: холодный серый
- основной акцент: глубокий зеленый
- дополнительный акцент: насыщенный оранжевый для CTA

Пример ролей цветов:
- `Background`: `#F5F4EF`
- `Surface`: `#FFFFFF`
- `Text Primary`: `#1F2328`
- `Text Secondary`: `#5B6470`
- `Primary Accent`: `#1F6B4F`
- `Action Accent`: `#E57A22`
- `Border`: `#D9DCDD`
- `Success`: `#2F8F5B`
- `Warning`: `#C77A12`
- `Danger`: `#B84A3A`

## 5. Типографика

Подход:
- выразительный современный гротеск для заголовков
- нейтральный, очень читабельный шрифт для интерфейса и карточек

Рекомендуемая логика:
- заголовки: плотные, уверенные, немного технические по характеру
- основной текст: спокойный, высокий уровень читаемости
- акцентные цифры и шаги: крупные, контрастные

## 6. Композиция и сетка

Рекомендуемый подход:
- чистая модульная сетка
- много воздуха между секциями
- выраженная иерархия блоков
- карточки средней плотности, без перегруза

Главный принцип:
- сначала задача и сценарий
- потом товар
- потом уточняющие характеристики

## 7. Основные UI-принципы

- быстрый вход через задачи ухода
- минимум лишних решений для пользователя
- на карточке товара сначала польза, потом детали
- важные предупреждения и способ применения всегда рядом
- связанные товары и следующие шаги показываются контекстно
- знания и инструкции должны быть визуально равноправны каталогу

## 8. Главная страница

Рекомендуемая структура:
1. `Hero` с оффером и быстрым выбором задачи ухода.
2. Блок популярных задач: кузов, салон, стекла, диски.
3. Хиты каталога.
4. Пошаговые комплекты.
5. База знаний и инструкции.
6. Блок консультации по подбору.
7. Преимущества магазина и доверительные блоки.

## 9. Каталог

Каталог должен выглядеть спокойно и функционально:
- чистые фильтры
- понятная сортировка
- карточки без перегруза
- быстрые действия: в избранное, в сравнение, в заявку

На мобильной версии:
- фильтры открываются в `drawer`
- основные действия доступны без лишнего скролла

## 10. Карточка товара

Карточка товара должна быть построена вокруг практического применения.

Порядок смысловых блоков:
1. Фото и название.
2. Для какой задачи подходит.
3. Основные преимущества.
4. Как использовать.
5. Предупреждения и совместимость.
6. FAQ.
7. Связанные товары.
8. Следующий шаг ухода.

## 11. Страницы задач ухода

На странице задачи ухода нужно показать:
- описание проблемы
- шаги решения
- рекомендуемый комплект товаров
- инструкции
- частые ошибки
- связанные статьи
- кнопку консультации

## 12. База знаний

Раздел базы знаний должен выглядеть как экспертный центр, а не как второстепенный блог.

Нужны:
- хорошие превью статей
- понятная навигация по темам
- связка статей с задачами ухода и товарами
- акцент на практической пользе

## 13. Мобильная версия

Мобильный интерфейс должен проектироваться не как уменьшенная копия десктопа, а как основной сценарий использования.

На мобильном приоритетны:
- крупные CTA
- быстрая навигация к задачам ухода
- фиксированные быстрые действия в карточке товара
- удобное сравнение и добавление в заявку
- минимальное число действий до консультации

## 14. Дизайн-система

На уровне дизайн-системы стоит сразу предусмотреть:
- токены цветов
- типографическую шкалу
- единый набор отступов
- состояния кнопок и полей
- карточки товара
- карточки задач ухода
- блоки предупреждений
- бейджи статуса заявки
- блоки шагов ухода

## 15. Образ бренда

`WeilShop` должен восприниматься как:
- современный
- аккуратный
- экспертный
- понятный даже новичку
- ориентированный на реальный результат ухода за автомобилем

Краткая формулировка:

`Технологичный магазин-консультант по автохимии с понятной подачей и чистым современным интерфейсом.`
