# Глоссарий WeilShop

Словарь терминов и определений для проекта WeilShop.

---

## 1. Предметная область (Автохимия)

### Автохимия
Специализированные химические средства для ухода за автомобилем: мойки, полироли, очистители, защитные составы.

### Задача ухода (CareTask)
Конкретная цель по уходу за автомобилем, например: «мойка кузова», «очистка салона», «уход за пластиком». Ключевая сущность для подбора товаров.

### Пошаговый комплект (CareKit)
Набор товаров, сгруппированных по шагам применения для решения конкретной задачи ухода.

**Пример:** Комплект для мойки кузова
1. Активная пена
2. Шампунь для ручной мойки
3. Воск-быстросохнущий
4. Микрофибра для сушки

### Совместимость поверхностей
Информация о том, для каких материалов безопасно использование средства (пластик, кожа, ткань, металл, стекло).

### Меры предосторожности
Предупреждения о безопасности при использовании средства: защита рук, вентиляция, запрет на смешивание с другими средствами.

### Объём (Volume)
Фасовка товара: 500 мл, 1 л, 5 л и т.д.

### SKU (Stock Keeping Unit)
Уникальный артикул товара для складского учёта.

### Бренд
Производитель автохимии: Koch Chemie, Grass, Shine Systems, Detail и др.

## 2. Пользователи и роли

### Пользователь (User)
Зарегистрированный посетитель сайта с личным кабинетом.

### Гость (Guest)
Незарегистрированный посетитель сайта.

### Администратор (Admin)
Сотрудник с доступом к административной панели.

### Super Admin
Администратор с полными правами: создание админов, назначение ролей, системные настройки.

### Catalog Manager
Администратор, отвечающий за товары, категории, бренды, задачи ухода.

### Content Manager
Администратор, отвечающий за статьи, инструкции, FAQ, SEO.

### Request Manager
Администратор, обрабатывающий заявки пользователей.

### Consultation Manager
Администратор, отвечающий на запросы по подбору автохимии.

### SEO / Merch Manager
Администратор, отвечающий за видимость товаров и коммерческую подачу.

## 3. Заказы и заявки

### Заявка (RequestOrder)
Запрос пользователя на покупку товаров без онлайн-оплаты. Формируется из корзины.

### Статусы заявки

| Статус | Описание |
|--------|----------|
| `NEW` | Новая заявка, ожидает обработки |
| `IN_REVIEW` | Администратор изучает заявку |
| `CONFIRMED` | Заявка подтверждена, комплектуется |
| `PROCESSING` | Заявка в работе (упаковка, передача в доставку) |
| `COMPLETED` | Заявка завершена, пользователь получил заказ |
| `CANCELLED` | Заявка отменена |

### История статусов (RequestStatusHistory)
Лог всех изменений статусов заявки с комментариями администратора.

### Корзина (Cart)
Временный список товаров, которые пользователь планирует заказать.

## 4. Контент

### Статья (Article)
Материал базы знаний: обзоры, советы, рекомендации по уходу.

### Инструкция (Instruction)
Пошаговое руководство по применению товара или решению задачи.

### FAQ (Frequently Asked Questions)
Ответы на часто задаваемые вопросы по товарам и задачам ухода.

### База знаний (Knowledge Base)
Раздел сайта со статьями, инструкциями и FAQ.

## 5. Функциональные сущности

### Избранное (Favorite)
Список товаров, сохранённых пользователем для быстрого доступа.

### История просмотров (ProductView)
Автоматически сохраняемая история просмотренных пользователем товаров.

### Сравнение (ProductComparison)
Инструмент для сравнения характеристик нескольких товаров.

### Консультация (ConsultationRequest)
Запрос пользователя на помощь в подборе автохимии от специалиста.

### Уведомление (Notification)
Сообщение пользователю о событиях: смена статуса заявки, ответ на консультацию.

### Каналы уведомлений

| Канал | Описание |
|-------|----------|
| `EMAIL` | Email-рассылка |
| `SMS` | SMS-сообщения (будущее) |
| `PUSH` | Push-уведомления в приложении (будущее) |
| `TELEGRAM` | Уведомления в Telegram-боте (будущее) |
| `IN_APP` | Внутренние уведомления в личном кабинете |

## 6. Технические термины

### API-first
Подход к разработке, при котором API проектируется до реализации frontend и является единым источником данных для всех клиентов.

### Slug
Человеко-понятный URL-идентификатор сущности: `mojka-kuzova`, `ochistka-salona`.

### JWT (JSON Web Token)
Стандарт токенной аутентификации. Состоит из access token и refresh token.

### Access Token
Короткоживущий токен (15 минут) для доступа к API.

### Refresh Token
Долгоживущий токен (7 дней) для обновления access token.

### Rate Limiting
Ограничение количества запросов от одного пользователя за единицу времени.

### CORS (Cross-Origin Resource Sharing)
Механизм разрешения запросов с других доменов.

### Prisma ORM
Инструмент для работы с базой данных через типобезопасный клиент.

### Migration
Версионирование схемы базы данных через SQL-скрипты.

### Seed
Начальное наполнение базы данных тестовыми или справочными данными.

### S3-compatible storage
Объектное хранилище, совместимое с Amazon S3 API (Yandex Cloud Storage, Selectel, MinIO).

## 7. SEO-термины

### Meta Title
Заголовок страницы для поисковых систем (50-70 символов).

### Meta Description
Краткое описание страницы для сниппета в поиске (150-300 символов).

### Meta Keywords
Ключевые слова страницы (устаревший тег, используется ограниченно).

### OG Tags (Open Graph)
Мета-теги для предпросмотра ссылок в соцсетях и мессенджерах.

### ЧПУ (Человеко-Понятный URL)
URL вида `/catalog/mojka-kuzova` вместо `/catalog?id=123`.

### Хлебные крошки (Breadcrumbs)
Навигационная цепочка: `Главная → Каталог → Мойка кузова → Шампунь`.

## 9. Безопасность

### bcrypt
Алгоритм хеширования паролей с «солью».

### Argon2
Современный алгоритм хеширования, победитель Password Hashing Competition.

### HTTPS
Защищённый протокол передачи данных через SSL/TLS.

### XSS (Cross-Site Scripting)
Атака через внедрение вредоносного JavaScript-кода.

### CSRF (Cross-Site Request Forgery)
Атака через выполнение действий от имени пользователя без его ведома.

### SQL Injection
Атака через внедрение SQL-кода в запросы к базе данных.

### 152-ФЗ
Федеральный закон «О персональных данных» (Российская Федерация).
