# User Flows WeilShop

## 1. Главная страница

1. Пользователь заходит на сайт.
2. Видит основные категории, задачи ухода и популярные товары.
3. Переходит в каталог, задачу ухода или базу знаний.

## 2. Каталог

1. Пользователь открывает каталог.
2. Фильтрует товары по категории, бренду и задаче ухода.
3. Переходит в карточку товара.
4. Добавляет товар в избранное или корзину.

## 3. Страница задачи ухода

1. Пользователь выбирает проблему или задачу.
2. Читает описание и пошаговую инструкцию.
3. Изучает рекомендованные товары.
4. Добавляет товары в корзину.

## 4. Карточка товара

1. Пользователь изучает описание товара.
2. Смотрит способ применения и меры предосторожности.
3. Переходит к инструкции или связанным материалам.
4. Добавляет товар в избранное, сравнение или корзину.

## 5. Оформление заявки

1. Пользователь открывает корзину.
2. Проверяет список товаров.
3. Заполняет имя, `email` или телефон, комментарий.
4. Отправляет заявку.
5. Получает номер заявки.

## 6. Личный кабинет

1. Пользователь авторизуется.
2. Открывает список своих заявок.
3. Переходит в карточку заявки.
4. Видит текущий статус и историю изменений.

## 7. Сравнение товаров

1. Пользователь добавляет несколько товаров в сравнение.
2. Открывает страницу сравнения.
3. Смотрит различия по объему, назначению, способу применения и характеристикам.
4. Выбирает подходящий товар и добавляет его в заявку или избранное.

## 8. Консультация по подбору

1. Пользователь открывает форму консультации.
2. Указывает задачу ухода, проблему и при необходимости конкретные товары.
3. Отправляет запрос на консультацию.
4. Видит статус консультации в личном кабинете.

## 9. Работа администратора с заявкой

1. Администратор получает новую заявку.
2. Открывает карточку заявки в админке.
3. Меняет статус заявки.
4. При необходимости добавляет внутренний комментарий.
5. Пользователь видит обновленный статус в личном кабинете.

## 10. Работа администратора с консультацией

1. Администратор получает новый запрос на консультацию.
2. Открывает карточку консультации.
3. Назначает ответственного и подготавливает рекомендацию.
4. Меняет статус консультации.
5. Пользователь получает уведомление и видит ответ в личном кабинете.
